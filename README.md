## 📝 Descrição do Projeto
Este projeto faz parte do Exercício 02 da disciplina de Interface e Jornada do Usuário. O objetivo foi desenvolver um protótipo de média fidelidade para um sistema complexo de segurança e validação de ID.

## 🚀 Fluxos Principais (Caminhos do Usuário)
Conforme solicitado no briefing, o sistema permite realizar a tarefa principal por dois caminhos:
1. **Caminho A:** Acesso direto via Dashboard através dos cards de "Informações de Acessos".
2. **Caminho B:** Navegação via Menu Lateral utilizando a seção de "Logs" e "Visualização Geral".

## 🛠️ Critérios de Design Aplicados
- **Arquitetura da Informação:** Organização em camadas de acesso (Admin, Normal, Pro).
- **Hierarquia Visual:** Uso de cores contrastantes para status de segurança.
- **Dados Densos:** Tabelas otimizadas para visualização de logs e detalhes de conta.
